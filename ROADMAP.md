## 📊 기술 스택 진화 계획

```
Frontend: Vanilla JS + HTML + CSS

Build: 없음 (직접 실행)
```

## 📝 의사결정 로그

### 왜 이 기술을 선택했는가?

#### localStorage vs IndexedDB
**결정**: v1.x에서는 localStorage, v1.6부터 IndexedDB
**이유**:
- ✅ localStorage: 간단, 빠른 프로토타입
- ❌ localStorage: 용량 제한 (5MB)
- ✅ IndexedDB: 대용량 (50MB+), 구조화된 데이터
- ❌ IndexedDB: 복잡한 API

**결론**: 점진적 마이그레이션

#### Vanilla JS vs React
**결정**: v1.x Vanilla, v2.x React
**이유**:
- ✅ Vanilla: 빠른 시작, 의존성 없음
- ❌ Vanilla: 대규모 시 유지보수 어려움
- ✅ React: 컴포넌트 재사용, 생태계
- ❌ React: 학습 곡선, 번들 크기

**결론**: MVP는 Vanilla, 확장 시 React

#### Supabase vs Firebase vs Custom Backend
**결정**: Phase 3에서 Supabase
**이유**:
- ✅ Supabase: PostgreSQL, 오픈소스, 저렴
- ❌ Firebase: NoSQL (관계형 데이터 부적합)
- ❌ Custom Backend: 개발/유지보수 부담

**결론**: Supabase 선택 (관계형 데이터 + 빠른 구축)
