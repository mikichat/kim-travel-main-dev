# 생년월일 19xx→20xx 오류 최종 수정

## 🚨 발견된 문제

**일심회 그룹 입력 시 19xx년생들이 20xx년으로 잘못 저장됨**

예시:
- 1985년생 → 2085년생으로 저장 → 만 -60세 ❌
- 1975년생 → 2075년생으로 저장 → 만 -50세 ❌

## 🔍 근본 원인

**3가지 문제점 발견:**

### 1. ddMMMyy 형식 문자열 (Line 881-911)
```javascript
// 예: "15MAY85" 입력 시
// ❌ 이전: 무조건 2085년으로 변환
// ✅ 수정: 현재 연도 기준 판단 → 1985년
```

### 2. Excel 숫자 형식 날짜 (Line 929-992)
```javascript
// Excel에서 날짜가 숫자로 저장된 경우
// ❌ 이전: 1900년대를 2000년대로 강제 변환
// ✅ 수정: dateType에 따라 올바르게 처리
```

### 3. Date 객체 처리 (Line 994-1022)
```javascript
// Date 타입으로 변환된 경우
// ❌ 이전: 1900년대를 2000년대로 강제 변환
// ✅ 수정: dateType에 따라 올바르게 처리
```

## ✅ 수정 완료 사항

### 1. formatDate 함수 완전 개선

**파일:** `group-roster-manager-v2 (3).html`

**수정 내용:**
- ✅ dateType 파라미터 추가 (기본값: 'birth')
- ✅ 문자열 2자리 연도: 현재 연도 기준 판단
- ✅ Excel 숫자 날짜: dateType별 처리
- ✅ Date 객체: dateType별 처리

### 2. 수정된 로직

#### 생년월일 (dateType='birth')
```javascript
// 현재 2025년 기준
'85' → 85 > 25 → 1985년 ✅
'99' → 99 > 25 → 1999년 ✅
'00' → 0 <= 25 → 2000년 ✅
'24' → 24 <= 25 → 2024년 ✅
```

#### 여권 만료일 (dateType='passport')
```javascript
// 항상 미래 날짜 (2000년대)
'30' → 2030년 ✅
'50' → 2050년 ✅
```

## 🔧 잘못 저장된 일심회 데이터 수정 방법

### 방법 1: 자동 수정 도구 사용 (권장)

**파일:** `fix-ilsimhoe-birthdate.html` (**이미 열림!**)

**사용 방법:**
1. ✅ "현재 데이터 확인" 버튼 클릭
   - 잘못된 생년월일이 빨간색으로 표시됨
   - 나이가 음수인 경우 = 2000년대로 잘못 저장됨

2. ✅ "자동 수정 실행" 버튼 클릭
   - 2085년 → 1985년 (100년 빼기)
   - 2075년 → 1975년 (100년 빼기)
   - localStorage에 자동 저장됨

3. ✅ 단체명단 관리 페이지로 이동
   - 페이지 새로고침 (F5)
   - 일심회 그룹 열기
   - 생년월일 확인
   - **저장 버튼 클릭** (중요!)

4. ✅ 완료!

### 방법 2: 엑셀 재업로드

1. 일심회 엑셀 파일 준비
2. 단체명단 관리에서 일심회 그룹 선택
3. "엑셀 업로드" 버튼 클릭
4. 파일 선택 후 업로드
5. **수정된 formatDate 함수가 자동으로 올바르게 처리**
6. 저장

### 방법 3: 수동 수정

각 멤버의 생년월일을 하나씩 수정
