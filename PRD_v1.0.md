## 📊 Executive Summary

### 프로젝트 개요
전주 소재 **"여행세상"** 여행사의 **모든 여행 업무**(상담 + 항공 + 출입국 + 일정 관리)를 효율적으로 처리할 수 있는 웹 기반 통합 시스템

**팀 구성**: 2-3명 (모든 직원이 전 업무 담당)
**주요 고객층**: 중장년층 (50-70대) - 교회 단체, 동호회, 은퇴자 모임

### 핵심 문제
- ✅ **현재 80% 완성**: 기본 기능은 작동하지만 **디자인과 업무 흐름이 불편**
- ❌ **고객층 미스매치**: 중장년층 고객이 "글씨가 작아서 안 보여요" 불만
- ❌ **견적서/일정표 양식 불일치**: 기존 PDF 양식(123.pdf, 12.pdf)과 시스템 출력물이 달라서 혼란
- ❌ **반복 작업 비효율**: 비슷한 상품/견적서를 매번 새로 만듦
- ❌ **UI/UX 문제**: 상품관리 팝업, 사이드바 순서가 업무 흐름과 안 맞음
- ❌ **고객 이력 관리 부재**: 재방문 고객 응대 시 이전 상담 내역 확인 어려움

### 목표
**6개월 내에 "고객 상담 → 견적 + 일정표 생성 → 예약"까지 10분 내 완료 가능한 시스템 구축**
- ✅ 기존 브랜드 양식(123.pdf, 12.pdf) 그대로 시스템화
- ✅ 중장년층 고객도 쉽게 읽을 수 있는 문서 (큰 글씨, 명확한 구조)
- ✅ 여행 전문가의 업무 효율 50% 향상

## 👥 사용자 페르소나

### Primary Persona: 여행 전문가 "김국진" (45세)

**배경**:
- 여행사 경력 15년 (전주 소재 "여행세상" 대표)
- 하루 20건 이상 고객 상담
- **모든 여행 업무 담당**: 상담 + 항공권 예약 + 출입국 서류 + 일정 관리 + 가이드 조율
- 주로 단체 여행(학교, 교회, 동호회) 전문
- Excel, 카카오톡, 전화가 주요 업무 도구

**주요 고객층**:
- 🎯 **중장년층 (50-70대)**: 교회 단체, 동호회, 은퇴자 모임
- 🎯 **학교/기업**: 수학여행, 워크샵

**하루 일과**:
```
09:00 - 고객 문의 확인 (카카오톡, 전화)
10:00 - 견적서 작성 (라오스 3박5일, 방콕 4박5일 등)
11:00 - 항공권 확인 및 예약 (GDS 시스템)
12:00 - 중식 (고객 미팅 겸)
14:00 - 고객 상담 (견적 설명, 수정 요청)
15:00 - 확정된 단체 명단 정리 + 출입국 서류 준비
16:00 - 현지 가이드/호텔 조율 (이메일, 전화)
17:00 - 추가 문의 응대
18:00 - 내일 일정 준비 + 항공편 재확인
```

**Pain Points**:
1. 😫 **견적서/일정표를 너무 많이 만듦** (하루 10-15개)
   - "라오스 3박5일" 상품이 거의 비슷한데 매번 새로 입력
   - 일정표 포맷이 매번 달라서 통일성 없음
2. 😫 **고객 이력이 없어서** 지난 상담 내용 기억 안 남
   - "작년에 방콕 갔던 교회인데..." → 기록 찾기 어려움
3. 😫 **팝업창이 복잡해서** 입력하다가 헷갈림
   - 항공편 정보 입력 시 스크롤 많이 해야 함
4. 😫 **사이드바 메뉴 순서**가 업무 순서와 안 맞음
5. 😫 **중장년층 고객** 전화로 여러 번 물어봄
   - "견적서가 글씨가 작아서 안 보여요"

**Goals**:
- ✅ 10분 안에 견적서 + 일정표 완성하기
- ✅ 고객 이력 한눈에 보기 (재방문 고객 빠르게 대응)
- ✅ 반복 작업 최소화 (템플릿 활용)
- ✅ **전문적이고 보기 쉬운** 견적서/일정표로 신뢰 구축
- ✅ 중장년층 고객도 쉽게 읽을 수 있는 문서

**기술 수준**:
- 💻 Excel: 상급 (15년 경험)
- 💻 웹 사용: 중급
- 💻 단축키: 가끔 사용
- 💻 새 시스템 적응: 느린 편 (익숙해지면 빠름)

#### 신규 기능 #3: 일정표 생성 시스템 ⭐⭐⭐⭐⭐

**Why**: 견적서와 함께 일정표를 고객에게 제공해야 하는데 매번 수동으로 만듦

**현재 문제**:
```
❌ Word/Excel로 일정표 따로 작성 (시간 소요)
❌ 포맷이 매번 달라짐 (일관성 없음)
❌ 견적서와 정보가 불일치 (수동 입력 실수)
```

**개선안**:
```
✅ 기존 일정표 양식 그대로 시스템화 (12.pdf)
   - 여행세상 브랜드 일관성 유지
   - 중장년층이 읽기 쉬운 타임라인 구조

✅ 견적서 연동
   - 견적서에서 [일정표 생성] 버튼 클릭 → 자동 생성
   - 단체명, 일자, 여행지 자동 입력

✅ 일자별 타임라인 편집
   - 1일차, 2일차, 3일차... 카드 형식
   - 시간, 지역, 교통편, 세부 일정, 식사 입력

✅ 사진 첨부
   - 관광지 사진 업로드 (각 일자별 3장)
   - 호텔 사진 추가

✅ PDF 다운로드
   - 견적서 + 일정표 묶음 다운로드 가능
```

#### 1. 가독성 최우선 📖
```
✅ 글자 크기 증가
   - 본문: 16px (일반 14px → 16px)
   - 중요 정보: 18px (가격, 일자, 단체명)
   - 최소 글자 크기: 14px (작은 텍스트도)

   - 흐린 회색 텍스트 최소화
   - 빨간색 강조는 중요 정보만

✅ 줄 간격
   - Line-height: 1.8 (여유 있게)
   - 단락 간격: 24px
```

#### 2. 큰 클릭 영역 👆
```
✅ 버튼 크기
   - 높이: 48px (일반 44px → 48px)
   - 너비: 최소 120px
   - 버튼 간 간격: 12px (실수 클릭 방지)

✅ 입력 필드
   - 높이: 48px
   - 글자 크기: 16px

✅ 체크박스/라디오 버튼
   - 크기: 24px × 24px (일반 20px → 24px)
```

#### 3. 명확한 안내 🎯
```
✅ 액션 버튼 명확하게
   - "다음" → "다음 단계 (항공편)" (구체적으로)
   - "저장" → "견적서 저장하기"
   - "확인" → "견적서 다운로드"

✅ 필수 입력 강조
   - 빨간색 별표 (*)
   - "필수" 텍스트 추가
   - 에러 메시지 크고 명확하게

✅ 도움말 풍선
   - 물음표 아이콘 (?) 추가
   - 호버 시 설명 표시
```

#### 4. 단순한 인터페이스 🎨
```
✅ 한 화면에 정보 최소화
   - 위저드 방식 (단계별)
   - 스크롤 최소화

✅ 여백 충분히

✅ 복잡한 애니메이션 지양
   - 페이드인/아웃만 사용
```

## 🛠️ 기술 스택

### 현재 (v1.5)
```
Frontend: Vanilla JS + HTML + CSS
Storage: localStorage
Build: 없음
Deploy: 로컬 파일
```

### Phase 1-2 (v2.0 ~ v2.5)
```
Frontend: Vanilla JS (유지)
Storage: IndexedDB + localStorage
Build: Webpack (번들링, 최적화)
Deploy: 로컬 + 클라우드 백업 (Dropbox/Google Drive)
New:
  - jsPDF (견적서 생성)
  - Chart.js (대시보드 차트)
  - xlsx.js (Excel 처리)
```

### Phase 3 (v3.0)
```
Frontend: Next.js + React
Backend: Supabase (PostgreSQL)
Storage: Supabase Storage
Auth: Supabase Auth (팀원 계정)
Deploy: Vercel
Features:
  - 실시간 동기화 (팀원 간)
  - 클라우드 백업 (자동)
  - 모바일 접근 가능
```

**마이그레이션 전략**:
- Phase 1-2: Vanilla JS로 디자인/기능 완성
- Phase 3: 안정화 후 점진적 이전
  1. 데이터 구조 먼저 이전 (IndexedDB → Supabase)
  2. 화면 하나씩 React 컴포넌트로 변환
  3. 기존 시스템과 병행 운영 (1개월)
  4. 완전 전환
