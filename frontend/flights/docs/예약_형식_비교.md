# 예약 형식 비교 분석

## 📊 두 가지 예약 형식 비교

### 형식 1 (기존)
```
1.KIM/YONGKUMR   2.KO/KUKJINMR   3.CHOI/HANYOUNGMR
4.KIM/SANGHYUNMR   5.LEE/HAKRYONGMR   6.YOON/SEOKDEUKMR
7  KE 471 I 19FEB 4 ICNSGN HK6  0910 1250  19FEB  E  KE/DZ3LEF
8  KE 476 R 25FEB 3 SGNICN HK6  2350 0645  26FEB  E  KE/DZ3LEF
```

**특징:**
- 승객 번호: `1.`, `2.`, `3.` ...
- 이름 형식: `성/이름+호칭`
- 호칭: MR, MS, MSTR, MISS
- 항공편: 별도 라인에 표시

---

### 형식 2 (새로운)
```
1.1SO/JAEWANMR  2.1SHIN/HYOUNGSUKMR  3.1LEE/SEUNGKWONMR
 1 OZ 707V 25FEB 3 ICNCRK HK3  2130  0030   26FEB 4
                                               /DCOZ*D6L2WS /E
 2 OZ 708E 02MAR 1 CRKICN HK3  0135  0640  /DCOZ*D6L2WS /E
TKT/TIME LIMIT
```

**특징:**
- 승객 번호: `1.`, `2.`, `3.` ...
- 이름 형식: `숫자+성/이름+호칭` (예: `1SO/JAEWANMR`)
- 이름 앞에 추가 숫자 존재 (예: `1SO`)
- 호칭: MR, MS, MSTR, MISS
- 항공편: 별도 라인에 표시

---

## 🔍 주요 차이점

| 항목 | 형식 1 | 형식 2 |
|------|--------|--------|
| 이름 패턴 | `KIM/YONGKUMR` | `1SO/JAEWANMR` |
| 이름 앞 숫자 | ❌ 없음 | ✅ 있음 (1, 2 등) |
| 정규식 | `\d+\.([A-Z/]+)` | `\d+\.(\d*[A-Z][A-Z/]+)` |
| 파싱 후 처리 | 호칭만 제거 | 앞 숫자 + 호칭 제거 |

---

## 🛠️ 통합 파싱 전략

### 1단계: 정규식 매칭
```javascript
// 두 형식 모두 매칭하는 정규식
const nameRegex = /(\d+)\.(\d*[A-Z][A-Z/]+)/g;

// 매칭 예시:
// 형식 1: "1.KIM/YONGKUMR" → match[1]="1", match[2]="KIM/YONGKUMR"
// 형식 2: "1.1SO/JAEWANMR" → match[1]="1", match[2]="1SO/JAEWANMR"
```

### 2단계: 앞 숫자 제거
```javascript
let fullName = match[2];
fullName = fullName.replace(/^\d+/, '');

// 처리 결과:
// 형식 1: "KIM/YONGKUMR" → "KIM/YONGKUMR" (변화 없음)
// 형식 2: "1SO/JAEWANMR" → "SO/JAEWANMR" (1 제거)
```

### 3단계: 호칭 제거
```javascript
function cleanPassengerName(name) {
    return name.replace(/(MR|MS|MSTR|MISS)$/i, '').trim();
}

// 최종 결과:
// 형식 1: "KIM/YONGKU"
// 형식 2: "SO/JAEWAN"
```

## 🎯 구현 우선순위

1. **정규식 패턴 업데이트** (필수)
   - 기존: `/(\d+)\.([A-Z/]+)/g`
   - 신규: `/(\d+)\.(\d*[A-Z][A-Z/]+)/g`

2. **앞 숫자 제거 로직 추가** (필수)
   ```javascript
   fullName = fullName.replace(/^\d+/, '');
   ```

3. **테스트 케이스 작성** (권장)
   - 형식 1 단독 테스트
   - 형식 2 단독 테스트
   - 혼합 형식 테스트

## 📝 추가 고려사항

### 1. 항공편 파싱 차이
- 형식 1: `7  KE 471 I 19FEB 4 ICNSGN HK6  0910 1250  19FEB  E  KE/DZ3LEF`
- 형식 2: ` 1 OZ 707V 25FEB 3 ICNCRK HK3  2130  0030   26FEB 4`

**차이점:**
- 형식 2는 항공편 번호 뒤에 클래스 코드가 붙음 (`707V`)
- 형식 2는 추가 정보 라인이 있음 (`/DCOZ*D6L2WS /E`)

### 2. 도착 날짜 처리
- 형식 1: 도착 날짜가 명시됨 (`19FEB`)
- 형식 2: 도착 날짜가 명시됨 (`26FEB 4`)

### 3. PNR 정보
- 형식 1: `KE/DZ3LEF`
- 형식 2: `OZ*D6L2WS` (별도 라인)
