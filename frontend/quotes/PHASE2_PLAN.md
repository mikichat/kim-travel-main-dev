# Phase 2 Implementation Plan
# 일정표 페이지 편집 기능 추가

**날짜**: 2025-12-25
**상태**: Planning

---

## 1. Phase 2 목표

Phase 1에서 완성한 견적서 편집 시스템에 **일정표 페이지 편집 기능**을 추가합니다.

### 핵심 목표
- ✅ 일정표 페이지 구조 분석
- ✅ 일정표 데이터 구조 정의
- ✅ 일정표 편집 UI 구현
- ✅ 견적서 + 일정표 통합 관리
- ✅ 다중 페이지 PDF 출력

---

## 2. 현재 상황 분석

### 2.1 기존 파일 구조
```
itinerary/chain/can/
├── 25년11월중순 광저우 망산-1.html  (전체 문서 - 1페이지 견적서 + N페이지 일정표)
├── 25년11월중순 광저우 망산-1_tailwind.html  (Tailwind 버전)
├── 25년11월중순 광저우 망산-1_hd1.png  (헤더 로고)
├── 25년11월중순 광저우 망산-1_hd2.png  (푸터 로고)
├── 25년11월중순 광저우 망산-1_hd3.png  (일정표 페이지 2?)
├── 25년11월중순 광저우 망산-1_hd4.png  (일정표 페이지 3?)
├── ...
└── 25년11월중순 광저우 망산-1_hdf.png  (일정표 마지막 페이지?)
```

### 2.2 일정표 이미지 파일
- `hd3.png` ~ `hdf.png`: 일정표 페이지 이미지
- 현재: 이미지를 HTML에 배경으로 삽입
- 문제점: 이미지 형태라서 텍스트 수정 불가능

---

## 3. Phase 2 접근 방식

### 옵션 1: 이미지 기반 + 오버레이 텍스트 (빠른 구현)
**방법**:
- 기존 일정표 이미지를 그대로 사용
- 수정 가능한 텍스트만 이미지 위에 오버레이로 배치
- CSS positioning으로 정확한 위치 지정

**장점**:
- 빠른 구현 (2-3시간)
- 레이아웃 100% 보존
- 이미지 변환 불필요

**단점**:
- 제한적인 편집 (텍스트만 가능)
- 이미지 파일 의존성

**추천도**: ⭐⭐⭐ (Phase 2.1 초기 구현용)

---

### 옵션 2: HTML 재구성 (완전한 편집)
**방법**:
- 일정표 이미지를 분석하여 완전한 HTML/CSS로 재구성
- 모든 요소를 편집 가능하게 만듦
- Phase 1 견적서와 동일한 방식

**장점**:
- 완전한 편집 자유도
- 이미지 파일 불필요
- 일관된 시스템

**단점**:
- 구현 시간 길음 (8-12시간)
- 복잡한 레이아웃 재현 어려움

**추천도**: ⭐⭐⭐⭐⭐ (Phase 2.2 최종 구현용)

---

### 옵션 3: PDF 기반 문서 시스템 (미래 확장)
**방법**:
- PDF를 이미지로 변환하여 사용
- Puppeteer 등으로 HTML → PDF 생성

**장점**:
- 자동화 가능
- 품질 우수

**단점**:
- 서버 필요
- 복잡도 높음

**추천도**: ⭐⭐ (Phase 3 이후 고려)

---

## 4. Phase 2 단계별 구현 계획

### Phase 2.1: 일정표 페이지 분석 및 프로토타입 (2-3시간)
**목표**: 일정표 이미지 분석 및 간단한 편집 시스템 구축

#### TASK 2.1.1: 일정표 이미지 분석
- [ ] hd3.png ~ hdf.png 각 이미지 확인
- [ ] 일정표 구조 파악
  - 날짜별 일정
  - 식사 정보
  - 호텔 정보
  - 관광지 정보
- [ ] 수정 가능 항목 식별
  - 날짜/요일
  - 일정 제목
  - 식사 내용
  - 호텔 이름
  - 관광지 설명
- [ ] 레이아웃 측정
  - 텍스트 위치 (top, left)
  - 폰트 크기
  - 색상
