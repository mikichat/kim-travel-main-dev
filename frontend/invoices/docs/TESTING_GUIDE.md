## 📝 테스트 시나리오 1: Advanced Mode 인보이스 생성

### 1단계: 인보이스 편집기 열기
브라우저에서 http://localhost:5000/invoice 접속

### 2단계: 기본 정보 입력
- **수신**: 오태완
- **일자**: 2026-01-02
- **내역**: 2026년01월05일 태국 파타야 5일

### 3단계: 계산 모드 선택
- "고급 계산 (계약금/잔금)" 라디오 버튼 클릭

### 4단계: Advanced Mode 데이터 입력

**여행 경비 계산**:
- **1인당 요금**: 1,770,000원
- **총 인원**: 16명
- **총 여행경비**: 28,320,000원 (자동 계산됨)

**계약금**:
- **계약금 금액**: 10,000,000원
- **계약금 설명**: "출입원 17명 아시아나 항공 구입 비용"

**추가 비용 항목**:
1. "추가 항목 추가" 버튼 클릭
2. 항목명: "취소수수료"
3. 금액: 200,000원
4. 설명: "송금식대표님"
5. 타입: "더하기(+)"

**계산 결과 확인**:
- **잔금**: 18,520,000원 (파란색 카드로 표시됨)

### 5단계: 저장 버튼 클릭
- "저장" 버튼 클릭
- 성공 메시지: "인보이스가 저장되었습니다."
- 콘솔에 저장된 인보이스 정보 출력

### 6단계: 데이터베이스 확인
브라우저 개발자 도구(F12) → Console 탭에서 저장된 데이터 확인:

```javascript
{
  id: "...",
  invoice_number: "INV-20260102-...",
  calculation_mode: "advanced",
  base_price_per_person: 1770000,
  total_participants: 16,
  total_travel_cost: 28320000,
  deposit_amount: 10000000,
  balance_due: 18520000,
  additional_items: [...], // JSON 배열
  ...
}
```

  - 합계: 5,000,000원 (자동 계산)

### 5단계: 저장 버튼 클릭
- "저장" 버튼 클릭
- 성공 메시지 확인

## 🔍 검증 포인트

### ✅ Advanced Mode
1. [ ] 고급 계산 섹션이 표시됨
2. [ ] 실시간 계산이 작동함 (총 여행경비, 잔금)

4. [ ] 저장 시 유효성 검증 (필수 필드)
5. [ ] API 호출 성공 (201 Created)
6. [ ] 저장된 데이터에 calculation_mode: "advanced" 포함
7. [ ] additional_items가 JSON 문자열로 저장됨

2. [ ] 항목 추가/삭제 가능

5. [ ] API 호출 성공
6. [ ] 저장된 데이터에 calculation_mode: "simple" 포함

## 🐛 문제 해결

### 문제 1: "저장 중 오류가 발생했습니다"
**원인**: 서버가 실행되지 않았거나 API 엔드포인트 오류
**해결**:
```bash
cd backend
node server.js
```

### 문제 2: "수신자와 일자는 필수입니다"
**원인**: 필수 필드 미입력
**해결**: 수신자, 일자 필드 입력 확인

### 문제 3: Advanced Mode에서 "입력 오류" 알림
**원인**: 1인당 요금 또는 총 인원이 0
**해결**: 유효한 숫자 입력 (> 0)

### 문제 4: 콘솔에 CORS 오류
**원인**: 프론트엔드/백엔드 포트 불일치
**해결**:
- 프론트엔드: http://localhost:5000 (같은 포트에서 서빙됨)
- API_BASE_URL 확인: `in/js/invoice-editor.js:2`
