# 인보이스 시스템 개발 요약

## 개요
인보이스 생성 및 미리보기 시스템을 개발하고 개선한 내용을 정리합니다.

## 주요 기능

### 1. 기본 정보 입력
- **수신 거래처**: DB 데이터(드롭다운) 및 수동 입력 2가지 방식 지원
  - 드롭다운에서 저장된 거래처 선택 가능
  - 직접 입력 시 자동으로 거래처 목록에 저장 (localStorage 기반)
- **일자**: 페이지 로드 시 오늘 날짜로 자동 설정
- **설명**: 선택적 입력 필드

### 2. 항공 스케줄 연동
- **데이터 소스**: `/flight-schedule.html`의 FlightSyncManager를 통해 localStorage에서 데이터 로드
- **그룹 단위 선택**: 예약번호(PNR) 또는 단체명 기준으로 그룹 전체 구간을 한 번에 선택
  - 예: `[HUVHNQ] 마닐라 단체 - ICN → BKK (2025-01-09) [2구간]`
- **전체 구간 표시**: 선택 시 출발, 경유, 도착 등 모든 구간 정보 표시

### 3. 항목 및 금액 관리
- 동적 항목 추가/삭제 기능
- 단가 × 수량 자동 계산
- 총액 자동 계산

### 4. 은행 정보
- 은행 계좌 선택 기능
- 선택 시 은행명, 계좌번호, 예금주 정보 표시

### 5. 미리보기 기능
- 실시간 미리보기 지원
- 모든 구간의 항공 스케줄 정보 표시
- 항목별 금액 및 총액 표시
- URL 길이 제한 시 sessionStorage 사용

### 6. PDF 생성 기능 ⭐ NEW (2026-01-02)
- **클라이언트 측 PDF 생성**: jsPDF + html2canvas 사용
- **2단계 프로세스**:
  1. "PDF 생성" 버튼 클릭 → 미리보기 창 자동 열림
  2. 미리보기 창에서 "📄 PDF 다운로드" 버튼 클릭 → PDF 다운로드
- **자동 파일명 생성**: `인보이스_{수신자}_{날짜}.pdf`
  - 예: `인보이스_무주성립관광_20260102.pdf`
- **A4 한 페이지 자동 맞춤**:
  - 콘텐츠가 A4 높이 초과 시 자동 축소
  - 가로/세로 비율 유지
  - 중앙 정렬
- **여백 설정**:
  - 좌우 여백: 10mm
  - 상하 여백: 10mm
  - 실제 콘텐츠 영역: 190mm × 277mm

## 데이터 흐름

1. **항공 스케줄 로드**
   - FlightSyncManager → localStorage → 그룹 단위 변환 → 드롭다운 표시

2. **미리보기 데이터 전달**
   - URL 파라미터 (2000자 이하)
   - sessionStorage (2000자 초과)

3. **거래처 관리**
   - localStorage에 거래처 목록 저장
   - 입력 시 자동 저장

2. **공항 도시명 자동 변환**

3. **비고사항 및 체크박스**

   - 미리보기에서 체크박스 표시

1. **인보이스 생성**
   - `invoice-editor.html` 열기 (http://localhost:5000/invoice)
   - 수신 거래처 선택 또는 입력

2. **미리보기**
   - 미리보기 버튼 클릭

### 2026-01-02 (오후) - Advanced Mode 고급 계산 기능 ⭐ NEW

#### ✅ 완료된 작업 (Phase 1)

1. **계산 모드 선택 UI**
   - 간편 계산 / 고급 계산 라디오 버튼
   - 모드 전환 시 섹션 자동 표시/숨김
   - 위치: `invoice-editor.html` 계산 방식 카드

2. **Advanced Mode 입력 폼**
   - **여행 총경비**:
     - 1인당 요금 입력
     - 총 인원 입력
     - 총경비 자동 계산 (readonly)
   - **계약금 (선금)**:
     - 계약금 금액 입력
     - 계약금 설명 입력 (예: "출입원 17명 아시아나 항공 구입 비용")
   - **추가 비용 항목**:
     - 동적 추가/삭제 버튼
     - 항목명, 금액, 설명, 타입(추가/차감) 입력
     - 무제한 항목 추가 가능

3. **실시간 계산 로직** (`invoice-editor.js`)
   - 입력 시 즉시 자동 재계산
   - 계산식:
     ```javascript
     총경비 = 1인당요금 × 총인원
     추가비용합계 = Σ(추가항목) - Σ(차감항목)
     잔금 = (총경비 + 추가비용합계) - 계약금
     ```
   - 음수 잔금 시 빨간색 경고 표시
   - 실시간 계산 요약 카드 (그라데이션 배경)

4. **잔금 계산 표시**
   - 보라색 그라데이션 요약 카드
   - 총 여행경비, 추가 비용, 계약금, 잔금 분해 표시
   - 금액 자동 포맷팅 (₩28,320,000)

5. **CSS 스타일** (`invoice.css`)
   - 계산 모드 선택 스타일
   - 폼 행 그리드 레이아웃 (3열)
   - 추가 비용 항목 그리드 (5열)
   - 잔금 계산 요약 카드 스타일
   - 반응형 디자인 (모바일 지원)

6. **미리보기/PDF에 Advanced Mode 표시** (`invoice-preview.html`)
   - `collectFormData()` 함수 통합
   - Advanced Mode 데이터 수집 및 전달
   - 계산 내역 표시:
     ```
     여행경비용    ₩1,770,000 × 16명         = ₩28,320,000
     계약금       출입원 17명 아시아나...    = ₩10,000,000
     취소수수료    송금식대표님 (추가)       = ₩200,000
     ─────────────────────────────────────
     잔금        (₩28,320,000 + ₩200,000) - ₩10,000,000 = ₩18,520,000
     ```
   - Simple/Advanced Mode 자동 전환
   - 추가 비용 항목 동적 표시
   - 잔금 계산 공식 표시
   - PDF 생성 시 동일하게 표시

7. **유효성 검증**
   - 1인당 요금 필수 검증
   - 총 인원 필수 검증
   - 음수 잔금 경고
   - 미리보기/PDF 생성 전 자동 검증
