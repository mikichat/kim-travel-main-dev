# 통합 테스트 가이드

## 개요

이 문서는 여행사 계약 관리 시스템의 프론트엔드와 백엔드 통합 테스트를 수행하는 방법을 안내합니다.

## 사전 준비

### 1. 의존성 설치

```bash
# API 디렉토리로 이동
cd api

# Python 의존성 설치
pip install -r requirements.txt
```

### 2. 환경 변수 설정

`api/.env` 파일이 생성되어 있습니다. 기본적으로 SQLite를 사용하도록 설정되어 있습니다.

```env
# SQLite (개발/테스트용)
DATABASE_URL=sqlite:///./travel_agency.db
```

PostgreSQL을 사용하려면:

```env
DATABASE_URL=postgresql://postgres:password@localhost:5432/travel_agency
```

## 통합 테스트 실행

### 방법 1: 자동화된 테스트 스크립트 실행

#### 1단계: 백엔드 서버 시작

```bash
cd api
python main.py
```

또는:

```bash
cd api
uvicorn main:app --reload --port 8000
```

서버가 시작되면 다음 URL에서 확인할 수 있습니다:
- API 문서: http://localhost:8000/docs
- 헬스 체크: http://localhost:8000/health

#### 2단계: 통합 테스트 실행

새 터미널을 열고:

```bash
cd tests
python integration_test.py
```

### 방법 2: 수동 테스트

#### 1단계: 백엔드 서버 시작

```bash
cd api
python main.py
```

#### 2단계: 브라우저에서 프론트엔드 접속

```
http://localhost:8000/pages/group_list.html
```

#### 3단계: 수동 테스트 시나리오

##### 시나리오 1: 단체 생성 및 관리

1. **단체 목록 페이지 접속**
   - URL: `http://localhost:8000/pages/group_list.html`
   - 빈 목록 또는 기존 단체 목록 확인

2. **신규 단체 생성**
   - "신규 단체 생성" 버튼 클릭
   - 단체 정보 입력:
     ```
     단체명: 테스트 단체
     출발일: 2025-01-15
     도착일: 2025-01-22
     인원: 30명
     1인당 요금: 1,500,000원
     계약금: 5,000,000원
     ```
   - 자동 계산 확인:
     - 박수: 7박
     - 일수: 8일
     - 총액: 45,000,000원
     - 잔액: 40,000,000원
     - 잔액 완납일: 2025-01-08 (출발 7일 전)

3. **저장 및 확인**
   - "저장" 버튼 클릭
   - 성공 메시지 확인
   - 단체 목록으로 리다이렉트 확인
   - 새로 생성된 단체가 목록에 표시되는지 확인

##### 시나리오 2: 단체 대시보드 및 관리

1. **대시보드 접근**
   - 단체 목록에서 생성한 단체 행 클릭
   - 대시보드 페이지로 이동 확인

2. **기본 정보 탭 확인**
   - 요약 카드 확인 (기간, 인원, 총액, 상태)
   - 상세 정보 테이블 확인
   - 자동/수동 배지 확인

3. **일정 관리 탭**
   - "일정 관리" 탭 클릭
   - "일정 추가" 버튼 클릭
   - 일정 정보 입력:
     ```
     장소: 서울 → 파리
     교통: 대한항공 KE901
     시간: 10:00 출발, 14:30 도착
     일정:
       09:00 호텔 조식
       10:00 에펠탑 관광
       12:00 점심 식사
     식사: 조식, 중식
     숙박: 파리 힐튼 호텔
     ```
   - 저장 및 확인
   - day_no가 자동으로 1로 설정되었는지 확인
   - 일정 날짜가 출발일로 설정되었는지 확인

4. **취소 규정 탭**
   - "취소 규정" 탭 클릭
   - "취소 규정 추가" 버튼 클릭
   - 규정 입력:
     ```
     출발 전 며칠: 30
     위약금 비율: 10%
     설명: 출발 30일 전 취소
     ```
   - 저장 및 확인
   - 취소 기준일이 자동 계산되었는지 확인

5. **포함/불포함 탭**
   - "포함/불포함" 탭 클릭
   - "포함 항목 추가" 버튼 클릭
   - 항목 입력: "항공권 (이코노미 왕복)"
   - "불포함 항목 추가" 버튼 클릭
   - 항목 입력: "여행자 보험"
   - 저장 및 확인

6. **문서 출력 탭**
   - "문서 출력" 탭 클릭
   - "견적서 생성" 버튼 클릭
   - PDF 다운로드 확인
   - "계약서 생성", "일정표 생성" 테스트
   - "통합 문서 생성" 테스트

7. **상태 변경**
   - "기본 정보" 탭으로 돌아가기
   - "상태 변경" 버튼 클릭
   - "계약"으로 변경
   - 상태가 변경되었는지 확인

##### 시나리오 3: 자동 계산 및 수동 수정

1. **단체 수정 페이지 접근**
   - 대시보드에서 "기본 정보 수정" 버튼 클릭

2. **자동 계산 테스트**
   - 출발일 변경 → 박수, 일수, 완납일 자동 변경 확인
   - 인원 변경 → 총액, 잔액 자동 변경 확인
   - 1인당 요금 변경 → 총액, 잔액 자동 변경 확인

3. **수동 수정 테스트**
   - 총액의 "✏️" 버튼 클릭
   - 수동으로 값 변경
   - 배경색이 노란색으로 변경되었는지 확인
   - "수동 수정됨" 표시 확인
   - "↺" 버튼 클릭하여 자동 계산으로 되돌리기

4. **저장 및 확인**
   - 수정 사항 저장
   - 대시보드에서 수동 수정 플래그 확인

##### 시나리오 4: 검색 및 필터링

1. **단체 목록 페이지로 돌아가기**

2. **검색 테스트**
   - 단체명으로 실시간 검색
   - 검색 결과 확인

3. **필터 테스트**
   - 날짜 범위 필터 적용
   - 상태 필터 적용
   - "초기화" 버튼 테스트

4. **페이징 테스트**
   - 페이지 이동 버튼 테스트
   - 전체 개수 표시 확인

## API 엔드포인트 테스트

### Swagger UI를 통한 테스트

1. 브라우저에서 `http://localhost:8000/docs` 접속

2. 각 엔드포인트 테스트:

#### 단체 관리
- `POST /api/groups` - 단체 생성
- `GET /api/groups` - 단체 목록 조회 (필터링 테스트)
- `GET /api/groups/{id}` - 단체 상세 조회
- `PUT /api/groups/{id}` - 단체 수정
- `PATCH /api/groups/{id}/status` - 상태 변경
- `POST /api/groups/{id}/recalculate` - 재계산
- `DELETE /api/groups/{id}` - 단체 삭제

#### 일정 관리
- `POST /api/groups/{id}/itineraries` - 일정 추가
- `GET /api/groups/{id}/itineraries` - 일정 목록
- `PUT /api/groups/{id}/itineraries/{itinerary_id}` - 일정 수정
- `DELETE /api/groups/{id}/itineraries/{itinerary_id}` - 일정 삭제

#### 취소 규정
- `POST /api/groups/{id}/cancel-rules` - 취소 규정 추가
- `GET /api/groups/{id}/cancel-rules` - 취소 규정 목록
- `DELETE /api/groups/{id}/cancel-rules/{rule_id}` - 취소 규정 삭제

#### 포함/불포함
- `POST /api/groups/{id}/includes` - 항목 추가
- `GET /api/groups/{id}/includes` - 항목 목록
- `DELETE /api/groups/{id}/includes/{include_id}` - 항목 삭제

#### 문서 생성
- `POST /api/groups/{id}/documents/generate` - 문서 생성
- `GET /api/groups/{id}/documents/{doc_id}/download` - 문서 다운로드

## 문제 해결

### 서버가 시작되지 않는 경우

```bash
# 의존성 재설치
pip install -r api/requirements.txt

# 데이터베이스 초기화 (SQLite)
rm api/travel_agency.db
python api/main.py
```

### 프론트엔드가 백엔드에 연결되지 않는 경우

- CORS 설정 확인: `api/main.py`의 CORSMiddleware 설정
- 브라우저 콘솔에서 네트워크 오류 확인
- API 서버가 8000 포트에서 실행 중인지 확인

### PDF 생성이 실패하는 경우

```bash
# WeasyPrint 의존성 확인
pip install weasyprint Pillow

# Windows의 경우 GTK+ 설치 필요
# https://weasyprint.readthedocs.io/en/stable/install.html
```
