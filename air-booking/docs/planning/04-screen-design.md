# 04-screen-design.md — 화면 설계 (텍스트 기반)

**프로젝트명**: DB 통합 및 서버 일원화
**버전**: 1.0
**작성일**: 2026-04-01

---

## 개요

**설계 기준: 원데이터 하나 → 두 가지 뷰**

```
DB에는 PNR 원본 데이터만 저장 (중복 금지)
↓
항공편 변환기: 고객 친화적 표시 (도시명, 이미지)
예약장부:     업무 친화적 표시 (공항코드, 마감, 정산)
↓
두 화면 모두 같은 데이터를 다르게 표현
```

**기본 원칙**:
- 기존 기능 100% 유지
- 모바일 반응형 지원 (320px 이상)
- 접근성 고려 (WCAG 2.1 AA)

| PNR | 6자 영숫자, UNIQUE | "이미 존재하는 PNR" |
| 항공사 | 필수 | "항공사를 선택하세요" |
| 편명 | 필수, 숫자 4자 | "편명은 4자 숫자" |
| 출발지/도착지 | 3자 공항 코드 | "공항 코드가 잘못됨" |
| 날짜 | YYYY-MM-DD, 유효한 날짜 | "날짜 형식 오류" |
| 한글명 | 필수 | "한글명은 필수" |
| 영문명 | 필수, 대문자 | "영문명은 대문자로" |
| 여권 | 필수, 형식 | "여권 형식 오류" |

### 색상 대비

- 텍스트 vs 배경: 4.5:1 (일반 텍스트), 3:1 (큰 텍스트)
- 경고 메시지: 빨강 + 아이콘 (색상만으로 정보 전달 X)

- Tab 순서: 좌상→우하
- Enter: 저장/전송
- Esc: 모달 닫기

- 모든 버튼에 aria-label
- 테이블에 caption 태그
