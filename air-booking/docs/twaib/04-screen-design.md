# 04-screen-design.md — 화면 설계서

## 설계 원칙

### 1. 타겟 사용자: 40대+ 중심
- **큰 글씨**: 본문 16px+, 제목 24px+
- **높은 대비**: 텍스트와 배경 명확한 구분
- **넉넉한 여백**: 복잡한 인터페이스 회피
- **텍스트 중심**: 아이콘 최소화 (이해도 낮음)
- **직관적 네비게이션**: 1~2단계 이내

### 2. 컬러 팔레트
- **배경**: #FAFAF5 (아이보리, 부드러운 톤)
- **주 색상**: #1B3A5C (네이비, 신뢰성)
- **포인트**: #C8A45E (골드, 프리미엄감)
- **텍스트**: #333333 (진한 회색)
- **에러**: #E74C3C (빨강)
- **성공**: #27AE60 (초록)

### 3. 타이포그래피
- **글씨체**: 'Noto Sans KR', -apple-system, BlinkMacSystemFont, sans-serif
- **제목 (h1)**: 32px, 600 weight, 네이비
- **제목 (h2)**: 24px, 600 weight, 네이비
- **본문 (p)**: 16px, 400 weight, 회색
- **라벨 (label)**: 14px, 500 weight, 회색
- **버튼**: 16px, 500 weight, 흰색/네이비
- **작은 텍스트**: 12px, 400 weight

### 4. 버튼 스타일
- **Primary**: 네이비 배경(#1B3A5C) + 흰색 텍스트, 16px, 패딩 12px 24px
- **Secondary**: 흰색 배경 + 네이비 테두리 + 네이비 텍스트
- **Ghost**: 배경 없음 + 네이비 텍스트 + 호버 시 회색 배경
- **Disabled**: 회색 배경 + 밝은 회색 텍스트

### 5. 스페이싱
- **기본 유닛**: 8px
- **섹션 간 마진**: 32px
- **요소 간 마진**: 16px
- **내부 패딩**: 16px~24px

## 접근성 (A11y)

- **텍스트 대비**: WCAG AA 기준 (4.5:1 이상)
- **폼 레이블**: 모든 input에 label 요소 연결 (for/id)
- **키보드 네비게이션**: Tab 키로 모든 버튼/입력 필드 접근 가능
- **ARIA 레이블**: 모달, 토스트 메시지에 role/aria-label 추가
- **포커스 표시**: 포커스 받은 요소에 visible 아웃라인

```jsx
<label htmlFor="destination">여행지 *</label>
<input 
  id="destination" 
  type="text"
  aria-required="true"
  aria-label="여행지 입력"
/>

<button aria-label="AI 안내문 생성">
  AI 안내문 생성
</button>

<div 
  role="dialog" 
  aria-labelledby="modal-title"
  aria-modal="true"
>
  <h2 id="modal-title">항공편 불러오기</h2>
</div>
```
