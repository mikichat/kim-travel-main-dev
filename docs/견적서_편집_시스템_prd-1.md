# PRD (Product Requirements Document)
# 견적서 편집 시스템 - Phase 1

## 1. 제품 개요

### 1.1 제품명
여행 견적서 편집 시스템 (1페이지 - 견적서 정보)

### 1.2 목적
기존 25년11월중순광저우-1.html과 **동일한 레이아웃**을 유지하면서, 핵심 정보(단체명, 일자, 여행지, 1인 여행요금)를 **수정 가능**하게 만들어 이미지/PDF 출력이 가능한 시스템 구축

### 1.3 핵심 원칙
- ✅ **레이아웃 100% 유지**: 원본 HTML과 시각적으로 동일해야 함
- ✅ **이미지 출력 보장**: PDF/이미지로 변환 시에도 동일한 레이아웃 유지
- ✅ **데이터 분리**: 레이아웃과 데이터를 분리하여 수정 용이성 확보
- ✅ **단계적 구현**: 1페이지 완성 후 다른 페이지로 확장

---

## 2. 문제 정의 (Problem Statement)

### 2.1 현재 문제점
- 견적서 내용 수정 시 HTML 직접 수정 필요
- 단체명, 일자 등 반복 입력 필요 (기존 단체 데이터 활용 불가)
- 레이아웃 유지가 어려움
- 여러 견적서 관리 시 일관성 유지 어려움

### 2.2 해결 방향
- 핵심 데이터 항목만 입력/수정 가능한 구조
- 기존 단체 관리 시스템과 연동 준비
- 레이아웃 고정, 데이터만 교체
- 이미지/PDF 출력 품질 보장

---

## 3. 범위 (Scope)

### 3.1 Phase 1 범위 (본 문서)
**1페이지 - 견적서 정보 페이지**

**수정 가능 항목:**
1. **단체명** (group_name)
   - 현재 값: "중국 – 광저우 천저우 고의령 망산 PTY"
   - 추후: 기존 단체 DB에서 선택 가능

2. **일자** (travel_dates)
   - 현재 값: "2025년 11월 14일 ~ 11월 18일"
   - 추후: 기존 단체 DB에서 가져오기 가능

3. **여행지** (destination)
   - 현재 값: "중국 – 광저우 천저우 고의령 망산"

4. **1인 여행요금** (pricing)
   - 항공사명 (airline_name): "아시아나\n항공"
   - 출발편 (departure_flight): "인천 (08:40) - 광저우 (11:15)"
   - 귀국편 (return_flight): "광저우 (12:20) - 인천 (17:00)"
   - 옵션 (option_type): "노옵션"
   - 금액 (price_amount): "1,540,000원"

**고정 항목 (수정 불가):**
- 레이아웃 구조
- 헤더 (로고, 담당자 정보)
- 여행 조건 테이블 구조
- 스타일 및 색상

### 3.2 Phase 2 이후 범위 (추후 진행)
- 2페이지: 여행 일정표
- 3페이지: 여행 조건 상세
- 다중 페이지 통합 관리
- 완전한 단체 DB 연동

### 3.3 제외 범위 (Non-Scope)
- 여행 조건 테이블 내용 수정 (Phase 1에서는 고정)
- 일정표 이미지 변경 (Phase 1에서는 고정)
- 사용자 권한 관리
- 버전 관리 시스템

## 4. 사용자 정의 (User Persona)

### 4.1 주요 사용자
- **견적서 작성자**: 여행사 직원
- **승인자**: 관리자
- **고객**: 최종 PDF/이미지 수령자

### 4.2 사용자 니즈
1. 빠르게 견적서 생성하고 싶음
2. 기존 단체 정보를 재사용하고 싶음
3. 레이아웃 걱정 없이 내용만 수정하고 싶음
4. 출력 결과가 항상 동일하게 나오길 원함

---

## 5. 핵심 기능 요구사항 (Functional Requirements)

### 5.1 데이터 입력/수정 기능
**FR-1: 단체 정보 입력**
- 단체명 텍스트 입력 또는 기존 단체 선택
- 일자 직접 입력 또는 기존 단체 데이터 가져오기
- 여행지 텍스트 입력

**FR-2: 여행 요금 정보 입력**
- 항공사명 입력
- 출발편 정보 입력 (출발지, 출발시간, 도착지, 도착시간)
- 귀국편 정보 입력 (출발지, 출발시간, 도착지, 도착시간)
- 옵션 타입 입력
- 1인 요금 입력

**FR-3: 실시간 미리보기**
- 입력 즉시 견적서 레이아웃에 반영
- 원본과 동일한 레이아웃으로 표시

**FR-4: 저장 기능**
- 입력한 데이터 JSON 형태로 저장
- 저장된 데이터 불러오기

**FR-5: 출력 기능**
- PDF로 저장
- 이미지(PNG/JPG)로 저장
- 인쇄 기능

### 5.2 레이아웃 유지 기능
**FR-6: 고정 레이아웃**
- 25년11월중순광저우-1.html과 픽셀 단위 동일성 유지
- 폰트, 색상, 간격 완벽 재현
- 이미지 영역 고정

**FR-7: 반응형 미지원**
- A4 210mm 고정 폭
- 브라우저 크기 변경 시에도 레이아웃 고정

### 5.3 데이터 연동 준비
**FR-8: 단체 데이터 연동 준비**
- 단체 선택 시 단체명, 일자 자동 입력 (인터페이스만 준비)
- 추후 기존 단체 관리 DB와 연동 가능한 구조

## 6. 비기능 요구사항 (Non-Functional Requirements)

### 6.1 성능
- 데이터 입력 후 미리보기 업데이트: 0.1초 이내
- PDF 생성: 3초 이내

### 6.2 품질
- 레이아웃 정확도: 원본 대비 100% 일치

### 6.3 유지보수성
- HTML/CSS 수정 없이 데이터만 변경 가능
- 데이터 구조 문서화
- 코드 주석 필수

### 6.4 확장성
- Phase 2 (일정표 페이지) 연동 가능 구조
- 다중 페이지 통합 대비

## 7. 데이터 구조

### 7.1 수정 가능 데이터 모델

```json
{
  "quote_id": "string (견적서 ID)",
  "created_at": "datetime",
  "updated_at": "datetime",

  "group_info": {
    "group_id": "string (추후 단체 DB 연동용)",
    "group_name": "string (단체명)",
    "travel_dates": "string (여행 일자)",
    "destination": "string (여행지)"
  },

  "pricing": {
    "airline_name": "string (항공사명)",
    "departure_flight": "string (출발편 정보)",

    "option_type": "string (옵션)",
    "price_amount": "string (1인 요금)",
    "price_note": "string (요금 안내)"
  },

  "metadata": {
    "contact_info": "string (담당자 정보 - 고정)",
    "logo_path": "string (로고 경로 - 고정)",
    "template_version": "string (v1.0)"
  }
}
```

## 8. 사용자 플로우 (User Flow)

### 8.1 신규 견적서 생성
```
1. 견적서 편집 페이지 접속
2. [선택] 기존 단체 선택 OR 직접 입력
   - 단체 선택 시: 단체명, 일자 자동 입력
   - 직접 입력 시: 모든 항목 수동 입력
3. 여행지 입력
4. 1인 여행요금 정보 입력
   - 항공사명
   - 출발편 (출발지, 시간, 도착지, 시간)
   - 귀국편 (출발지, 시간, 도착지, 시간)
   - 옵션
   - 금액
5. 실시간 미리보기 확인
6. 저장
7. PDF/이미지 출력
```

### 8.2 기존 견적서 수정
```
1. 저장된 견적서 목록에서 선택
2. 데이터 로딩
3. 수정할 항목 변경
4. 실시간 미리보기 확인
5. 저장
6. PDF/이미지 출력
```

## 12. 제약 사항 및 리스크

### 12.1 제약 사항
- 레이아웃은 절대 변경 불가
- Phase 1에서는 1페이지만 구현
- 기존 단체 DB 구조 파악 필요 (연동 준비용)

### 12.2 리스크
- **리스크 1**: 레이아웃 재현 실패
  - 완화 방안: 원본 HTML 철저 분석, 픽셀 단위 비교

- **리스크 2**: PDF 출력 시 레이아웃 깨짐
  - 완화 방안: 출력 전용 CSS 작성, 다양한 브라우저 테스트

- **리스크 3**: 단체 DB 연동 지연
  - 완화 방안: Phase 1에서는 직접 입력만 지원
