# 골프 여행상품 랜딩페이지 & 관리 시스템 — 디자인 및 개발 방향서

> 작성일: 2026-03-05
> 대상: 골프 일정, 골프+관광 랜딩페이지 및 여행상품 관리 기능

## 1. 현황 분석

### 1.1 현재 보유 시스템 (관광 여행)

| 구분 | 상태 | 핵심 구성요소 |
|------|------|--------------|
| **관광 랜딩페이지** | ✅ 완료 | 표지 4종, 일정표(Standard/Luxury), 호텔정보, 국가정보, 도시관광지, 체크리스트, 견적서, 여정지도 |
| **여행상품 관리** (TourForm) | ✅ 완료 | 일자별 일정 CRUD, 엑셀/워드 임포트, 드래그 정렬, 브로슈어 자동 동기화 |
| **골프 일정** | ❌ 없음 | - |
| **골프+관광** | ❌ 없음 | - |

### 1.2 기존 아키텍처 요약

```
TourMaster ─── DailySchedule ─── ScheduleDetail
    │               │
    │               ├── hotelName, meals
    │               ├── images[], contentType
    │               └── details[]: region, transport, time, description
    │
    └── metadata.linkedTourId ──→ Brochure (양방향 동기화)
```

- **DB**: Prisma + SQLite
- **프론트엔드**: React + Vite, view state 기반 네비게이션
- **템플릿**: B5/A4 인쇄용 페이지 (PageHeaderFooter, CSS 변수 기반)
- **디자인 토큰**: `--color-orange`, `--color-teal`, `--color-navy`, Pretendard 폰트

## 2. 개발해야 할 3가지 상품 유형

### 유형 비교표

| 항목 | 관광 (기존) | 골프 전용 | 골프+관광 |
|------|-----------|----------|----------|
| 일정 구성 | 관광지 중심 | 라운딩 중심 | 오전 라운딩 + 오후 관광 |
| 핵심 정보 | 관광지, 식사, 호텔 | **골프장, 홀수, 티타임, 카트** | 골프장 + 관광지 혼합 |
| 숙박 | 호텔/리조트 | **골프텔/리조트** (골프장 인접) | 리조트 |

| 일정표 스타일 | 시간순 텍스트 | **라운딩 블록 + 자유시간** | 하이브리드 |

| 샘플 | 유형 | 일정 패턴 |
|------|------|---------|
| **베트남 다낭 3박5일** | 골프+관광 | DAY2 골프 → DAY3 관광 → DAY4 골프+관광(조 분리) |
| **중국 청도 4박5일** (화산/영해/천태/국제 72Holl) | 골프 전용 | DAY1~4 매일 라운딩 + DAY5 관광(맥주박물관) |
| **태국 파타야 3박5일** (Phoenix Gold/Pattaya CC/Burapha) | 골프 전용 | DAY2~4 매일 라운딩 |
| **중국 하문 4박5일** (오리엔트/해서/화도/남태무 72Holl) | 골프 전용 | DAY1~4 매일 라운딩 |

### 3.0.1 핵심 발견: 골프장 설명 블록 = 라운딩 정보 통합

**기존 관광 일정과의 가장 큰 차이**: 골프장 설명 블록이 일정표 안에 직접 삽입되며,
이 블록 안에 코스 설명 + 라운딩 정보(티타임, 홀수, 비용 포함/불포함)가 **하나로 통합**됨.

```
실제 샘플 패턴:
┌──────────────────────────────────────────────────────┐
│  • 가이드 미팅 후 화산 컨트리클럽 이동 (약 60분 소요)   │  ← 일반 일정
│                                                        │
│  ┌─ 골프장 설명 블록 ──────────────────────────────┐  │
│  │  중국 청도에서 두 번째로 오픈한 골프장으로서...   │  │  ← 코스 설명 (3~7줄)
│  │  전체 36홀 중 구코스 18홀, 신코스 18홀로 구성    │  │     홀수/야드/특징/난이도
│  └──────────────────────────────────────────────────┘  │
│                                                        │
│  • 석식 후 호텔 투숙                                    │  ← 일반 일정
│  [골프장 사진1] [사진2] [사진3]                          │  ← 이미지 3장
│  HOTEL: 청양 글로리아 OR 동급(2인실)                     │  ← 호텔
└──────────────────────────────────────────────────────┘
```

```
베트남 샘플 (골프+관광 혼합) — 더 상세한 라운딩 정보:
┌──────────────────────────────────────────────────────┐
│  골프조[2조 8명]                                       │  ← 조 편성
│  ▶ 호이아나 C.C. 18홀 라운 - 티업시간: 07:20 [자체수배] │  ← 라운딩 정보
│  - 그린피 & 카트 & 캐디피 포함                          │  ← 포함 내역
│  - 캐디팁(20$/1인/18홀) 불포함                          │  ← 불포함 내역
│                                                        │
│  관광조(3인) -호텔 자유시간                              │  ← 관광조 분리
└──────────────────────────────────────────────────────┘
```

### 3.0.2 식사 표기 — 골프 특화

| 패턴 | 예시 |
|------|------|
| 클럽하우스 식사 | `중: 클럽하우스`, `중: 클럽식` |
| 구체적 메뉴 | `무제한 삼겹살`, `소갈비찜`, `씨푸드`, `산동요리` |
| 불포함 강조 | **빨간색** `불포함` 텍스트 |
| 반제오정식 | `반제오정식` (베트남 현지 메뉴명) |

### 3.0.3 공통 부가 서비스

- `라운딩의 피로를 풀어주는 전신맛사지 90분 (팁별도)` — 거의 매일 포함
- `여행의 피로를 풀어줄 발맛사지 (2시간 / 팁별도)` — 볼드 + 빨간색 강조
- 이발소 탐방, 야간 시티투어 등 관광 옵션

## 5. 데이터 모델 설계 방향

### 5.1 DB 스키마 확장

기존 `TourMaster` / `DailySchedule`을 확장하는 방식으로, 새 테이블을 최소화합니다.

```prisma
// === 기존 모델 확장 ===

model TourMaster {
  // ... 기존 필드 유지
  tourType      String   @default("sightseeing")  // NEW: "sightseeing" | "golf" | "golf_sightseeing"
  golfCourses   GolfCourse[]                       // NEW: 연결된 골프장 목록
}

model DailySchedule {
  // ... 기존 필드 유지
  dayType       String   @default("sightseeing")   // NEW: "sightseeing" | "golf" | "mixed" | "travel" | "free"
  golfRound     GolfRound?                          // NEW: 라운딩 정보 (1:1)
}

// === 신규 모델 ===

model GolfCourseMaster {
  id            Int      @id @default(autoincrement())
  name          String                               // 골프장명 (영문/현지어)
  nameKo        String?                              // 골프장명 (한글)
  country       String                               // 국가
  city          String                               // 도시
  airport       String?                              // 최근접 공항 코드
  holes         Int      @default(18)                // 홀 수
  par           Int?                                 // 파
  yardage       Int?                                 // 야드
  courseType    String?                               // "링크스" | "파크랜드" | "리조트" 등
  rating        Float?                               // 평점 (5.0 기준)
  address       String?
  tel           String?
  url           String?
  description   String?                              // 코스 설명
  facilities    String?  @default("[]")              // JSON: ["드라이빙레인지", "퍼팅그린", "캐디"]
  images        String?  @default("[]")              // JSON: 코스 이미지 배열
  courseLayout   String?                              // 코스 레이아웃 이미지 URL
  greenFee      String?                              // 참고 그린피 정보 (텍스트)
  remarks       String?
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  golfRounds    GolfRound[]
  tourCourses   GolfCourse[]
}

model GolfCourse {
  id            Int      @id @default(autoincrement())
  tourId        Int
  courseId       Int
  playOrder     Int      @default(1)                 // 라운딩 순서

  tour          TourMaster       @relation(fields: [tourId], references: [id], onDelete: Cascade)
  course        GolfCourseMaster @relation(fields: [courseId], references: [id])
}

model GolfRound {
  id            Int      @id @default(autoincrement())
  scheduleId    Int      @unique
  courseId       Int?                                 // 마스터 연결 (선택적 — 없으면 수동 입력)
  courseName    String                                // 골프장명 (표시용, "◆Phoenix Gold Golf & Country Club◆")
  courseNameKo  String?                               // 골프장명 한글 ("피닉스 골드 골프 & 컨트리클럽")
  description   String?                               // 코스 설명문 (3~7줄, 일정표 내 표시)
  teeTime       String?                              // "07:20", "Shotgun 7:00", "미정"
  holes         Int      @default(18)                // 18홀 / 27홀 / 36홀
  includedItems String?                              // 포함 내역: "그린피 & 카트 & 캐디피 포함"
  excludedItems String?                              // 불포함 내역: "캐디팁(20$/1인/18홀) 불포함"
  groupInfo     String?                              // 조 편성: "골프조 [2조 8명]"
  sightseeingGroupInfo String?                       // 관광조 정보: "관광조(3인) -호텔 자유시간"
  remarks       String?                              // 비고 (드레스코드 등)
  images        String?  @default("[]")              // JSON: 골프장 사진 배열 (3장)

  schedule      DailySchedule    @relation(fields: [scheduleId], references: [id], onDelete: Cascade)
  course        GolfCourseMaster? @relation(fields: [courseId], references: [id])
}
```

### 5.2 핵심 설계 원칙

1. **기존 구조 재활용**: `TourMaster` → `tourType` 필드 추가만으로 상품 유형 구분
2. **DailySchedule 확장**: `dayType` + 선택적 `GolfRound` 연결
3. **골프장 마스터 분리**: `GolfCourseMaster`로 재사용 가능한 골프장 데이터베이스
4. **Brochure 동기화 유지**: 기존 `metadata.itineraryDays`에 골프 정보 포함

## 6. 골프 견적서 — `GolfQuotation.tsx` (신규)

> 기존 `TravelQuotation.tsx`와 별도 컴포넌트로 생성. 샘플 `samples/golf/1.jpg` 기반.

### 6.1 기존 견적서 vs 골프 견적서 비교

| 영역 | 관광 견적서 (`TravelQuotation`) | 골프 견적서 (`GolfQuotation`) |
|------|------------------------------|----------------------------|
| **1인 여행요금** | 금액 카드 (PriceItem 배열) | **항공사 + 스케줄 + "GOLF" 라벨 + 금액** 한줄 |
| **여행 조건** | 항공료/숙박/식사/교통편/관광/보험/세금 (7개) | 항공료/숙박/식사/교통편/**골프**/보험/세금 (8개) |
| **골프 조건** | 없음 | `전 일정에 명시된 골프 비용 포함` + 캐디팁/카트 조건 (빨간색) |
| **비고 사항** | 일반 | `4명 이상 행사 조건`, `단독 요금 진행 조건`, `VAT 별도` |
| **옵션/쇼핑** | 있음 | 없음 (골프 상품은 해당 없음) |

### 6.3 구현 방식

- **DB 변경**: `QuotationMaster`에 `quotationType String @default("travel")` 필드 추가 (`"travel"` | `"golf"`)
- **기존 conditions 배열 활용**: 골프 조건은 `{ label: "골프", value: "전 일정에 명시된 골프 비용 포함", note1: "캐디팁 150元 별도 / 전동카 2인 1카트 포함" }` 형태로 저장
- **1인 여행요금 레이아웃**: 기존 PriceItem 카드 대신 **항공스케줄 + GOLF 라벨 + 금액** 한줄 레이아웃
- **App.tsx 분기**: `quotationType === 'golf'` → `<GolfQuotation>`, 아니면 `<TravelQuotation>`
- **QuotationManager**: 생성 시 유형 선택 → 골프 선택하면 기본 conditions에 "골프" 행 포함

## 10. 컬러 & 디자인 시스템 확장

### 10.1 골프 테마 컬러 추가

기존 CSS 변수(`variables.css`)에 골프 전용 색상을 추가합니다.

```css
/* 골프 테마 — 기존 변수에 추가 */
--color-golf-green: #2d6a4f;        /* 메인 골프 그린 */
--color-golf-green-light: #d8f3dc;  /* 라운딩 블록 배경 */
--color-golf-fairway: #52b788;      /* 서브 액센트 */
--color-golf-sand: #f4e8c1;         /* 벙커/모래 색상 (선택적 포인트) */
```

### 10.2 상품 유형별 색상 적용 규칙

| 요소 | 관광 (기존) | 골프 | 골프+관광 |
|------|-----------|------|----------|
| 일정표 헤더 | `--color-teal` | `--color-golf-green` | `--color-golf-green` |
| 일자 번호 | `--color-orange` | `--color-orange` (유지) | `--color-orange` (유지) |
| 라운딩 블록 배경 | - | `--color-golf-green-light` | `--color-golf-green-light` |
| 골프장 카드 | - | `--color-golf-green` 보더 | `--color-golf-green` 보더 |
| 관광 일정 | 기본 | - | 기본 (기존 유지) |

## 11. 개발 페이즈 제안

### Phase 1: 데이터 기반 (1주)
1. Prisma 스키마 확장 (`tourType`, `dayType`, `GolfCourseMaster`, `GolfRound`, `GolfCourse`)
2. 마이그레이션 실행
3. 골프장 마스터 API (CRUD)
4. 투어 API 확장 (`tourType` 필터, 골프 정보 포함)

### Phase 2: 골프장 마스터 관리 UI (1주)
1. `GolfCourseManager.tsx` — 목록/검색/CRUD
2. AI 코스 설명 생성 (기존 Gemini 연동 활용)
3. 이미지 검색 연동 (기존 `imageSearch.service.ts` 재활용)

### Phase 3: TourForm 확장 (1주)
1. 상품 유형 선택 UI
2. 골프장 선택 패널
3. 일자별 일정 입력 — `dayType` + 라운딩 정보 토글
4. 기존 엑셀 임포트에 골프 정보 컬럼 대응

### Phase 4: 브로슈어 컴포넌트 (1주)
1. `GolfCourseCard.tsx` — 골프장 상세 카드 페이지
2. `ItineraryLuxury.tsx` 확장 — 라운딩 블록 렌더링
3. `TravelChecklist.tsx` 확장 — 골프 준비물 섹션
4. Brochure 미리보기 조합 — `tourType` 기반 페이지 구성 분기

### Phase 5: 동기화 & 마무리 (3일)
1. Tour ↔ Brochure 동기화에 골프 정보 포함
2. Brochure Wizard 스텝 확장 (골프장 선택 스텝)
3. PDF 출력 검증 (B5/A4)
4. 테스트 코드 추가

## 12. 필요한 샘플 자료 요청

문서 작성 및 개발 시 다음 자료가 있으면 큰 도움이 됩니다:

| # | 자료 | 용도 | 비고 |
|---|------|------|------|
| 1 | **골프 투어 일정표 샘플** (엑셀/워드/PDF) | 골프 일정 파서 개발, 일정표 레이아웃 확정 | 실제 사용하시는 형식 |
| 2 | **골프장 사진** (코스 전경, 클럽하우스 등) | 골프장 카드 디자인 검증 | 2~3개 골프장 |
| 3 | **골프 견적서 샘플** | 견적서 항목 차이 확인 (그린피, 캐디비 등) | 기존 견적서와 다른 점 있으면 |
| 4 | **골프+관광 혼합 일정표 샘플** | 하이브리드 일정 레이아웃 확정 | 오전 라운딩 + 오후 관광 |
