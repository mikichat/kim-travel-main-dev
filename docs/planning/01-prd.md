# PRD: 자유여행 예약 내역 이미지 생성 시스템

**문서 버전**: 1.0 | **작성일**: 2026-02-18 | **상태**: Active (MVP Phase 1)

---

## MVP 캡슐

| 항목 | 내용 |
|------|------|
| **목표** | 자유여행 고객에게 예약 내역을 모바일 최적화된 이미지로 전달 |
| **페르소나** | 여행사 직원(입력), 자유여행 고객(열람) |
| **성공 지표** | 고객에게 예약 내역 이미지 전달 완료율 70% 이상 |
| **입력 지표** | 에디터 입력 완료 → 이미지 저장 건수 |
| **Top 리스크** | html2canvas 긴 페이지 이미지 저장 품질 |
| **완화 방안** | 섹션별 개별 이미지 저장 옵션 제공 |

---

## 1. 개요

### 1.1 배경 및 문제
- **현상**: 자유여행 고객은 항공사, 호텔, 렌트카 회사 등 여러 채널에서 각각 예약 확인
- **문제**: 여행사 직원은 통합 예약 문서 제공 불가능 → 고객 만족도 저하
- **근본원인**: 기존 PDF 생성 방식이 복잡하고 확장성이 낮음

### 1.2 해결책
- **에디터 페이지**: 여행사 직원이 자유여행 예약 내역 통합 입력 (FEAT-1: 통합 뷰)
- **미리보기 페이지**: 고객 열람용 모바일 최적화 (FEAT-2: 이미지 저장)
- **데이터 관리**: localStorage 저장 + URL 파라미터 전달 (FEAT-3: 저장/불러오기)

### 1.3 제품 비전
> 여행사가 한 번의 입력으로 여행 고객에게 예약 **전체**를 **모바일 이미지**로 전달하는 경험

---

## 2. 범위 (In-Scope)

### 2.1 핵심 기능 (FEAT)

#### FEAT-1: 예약 내역 통합 뷰
**에디터 페이지에서 8개 섹션 입력/관리**

- **기본정보**: 수신인, 발신인, 작성일, 여행지, 여행기간
- **항공편**: 탑승객(색상 배정), 편명, 날짜, 출발/도착 공항·시간
- **숙박**: 숙소명, 체크인일, 박수, 예약자
- **렌트카**: 차종, 보험정보, 예약자
- **골프**: 골프장명, 날짜, 티오프시간, 예약자
- **커스텀 섹션**: 타입(투어/식사/입장권/액티비티/교통/기타), 항목명, 날짜, 메모, 예약자
- **결제정보**: 총액, 은행/계좌/예금주, 결제상태
- **회사정보**: 회사명/대표/주소/전화/팩스/도장이미지, 담당자

**특징**:
- 항목별 추가/삭제 가능 (기본정보, 결제, 회사 제외)
- 섹션별 토글 체크박스 (표시/숨김)
- 실시간 미리보기

#### FEAT-2: 이미지 저장 및 전달
**미리보기 페이지에서 이미지 변환**

- 전체 1장 저장 (html2canvas, PNG)
- 섹션별 개별 저장 (체크박스 선택)
- 모바일 최적화 (가로/세로 비율)
- 인쇄 지원 (print CSS)

#### FEAT-3: 데이터 저장/불러오기

**여행사 직원**: localStorage (freetravel_saves_v1)
- 입력 데이터 5초 이내 자동 저장
- 저장 목록 최대 10개 (이전 저장 자동 삭제)
- 불러오기로 상태 완전 복원

**고객 공유**: LZString 압축 → URL 파라미터
- 고객이 URL 접근 시 데이터 자동 로드
- 모바일(iOS, Android) 오프라인 열람 가능

## 4. 기술 스택

- **프론트엔드**: HTML + Tailwind CSS (CDN) + Vanilla JS
- **데이터 저장소**: localStorage (freetravel_saves_v1), LZString 압축
- **이미지 생성**: html2canvas (lazy load)
- **아이콘**: Material Symbols Outlined + Font Awesome
- **폰트**: Noto Sans KR + Inter (Google Fonts)
- **UI 패턴**: preview-mobile.html 디자인 톤 기반

## 5. 사용자 스토리

### 여행사 직원

**Story 1**: 예약 내역 신속 입력
```
AS A 여행사 직원
I WANT TO 에디터에서 한 번에 고객의 모든 예약 정보를 입력할 수 있고
I WANT TO 자동으로 저장되어
SO THAT I CAN 여러 고객의 예약을 효율적으로 관리할 수 있다
```

**Story 2**: 미완료 작업 복구
```
AS A 여행사 직원
I WANT TO 미완료 작업을 저장했다가 나중에 복원할 수 있고
I WANT TO 여러 고객 예약을 순차적으로 처리할 수 있기를
SO THAT I CAN 작업 중단/재개 시 진행 상태를 완벽하게 유지할 수 있다
```

**Story 3**: 최종 확인 후 고객 공유
```
AS A 여행사 직원
I WANT TO 미리보기에서 최종 확인하고 고객에게 공유할 준비를 할 수 있기를
SO THAT I CAN 정확한 예약 정보를 고객에게 전달할 수 있다
```

### 자유여행 고객

**Story 4**: 통합 예약 정보 열람
```
AS A 자유여행 고객
I WANT TO 여행사에서 받은 링크로 모든 예약 내역을 한 곳에서 확인할 수 있고
I WANT TO 모바일에서도 완벽하게 보일 수 있기를
SO THAT I CAN 항공, 호텔, 렌트카 등을 일일이 확인하는 번거로움을 줄일 수 있다
```

**Story 5**: 예약 이미지 저장
```
AS A 자유여행 고객
I WANT TO 예약 내역을 이미지로 저장해서 언제든 확인할 수 있기를
SO THAT I CAN 오프라인에서도 예약 정보를 참고할 수 있다
```

**Story 6**: 인쇄
```
AS A 자유여행 고객
I WANT TO 예약 내역을 출력할 수 있기를
SO THAT I CAN 오프라인으로 소지할 수 있다
```
