# PRD: Travel World CMS

## MVP 캡슐

| 항목 | 내용 |
|------|------|
| **목표** | 여행 데이터 입력 → PDF 브로슈어 출력 올인원 도구 |
| **페르소나** | 여행사 대표/관리자 |
| **핵심 기능** | FEAT-2: 일정표 편집기 |
| **성공 지표** | 브로슈어 제작 시간 30분 → 5분 이내 |
| **입력 지표** | 월간 브로슈어 발행 건수, 평균 제작 시간 |
| **비기능 요구** | 데스크톱 웹 전용, 하이브리드 저장 |
| **Out-of-scope** | 모바일 앱, 결제 시스템, 외부 판매 |
| **Top 리스크** | 드래그앤드롭 UI 구현 복잡도 |
| **완화 방안** | React-DnD 라이브러리 활용 |
| **다음 단계** | 일정표 편집기 프로토타입 개발 |

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
**Travel World CMS**

### 1.2 한 줄 요약
여행 데이터 입력부터 PDF 브로슈어 출력까지 한번에 처리하는 올인원 도구

### 1.3 배경 및 동기
- 현재 여행사에서는 브로슈어 제작 시 **엑셀, 한글, 포토샵** 등 여러 도구를 오가며 작업
- 매번 수동으로 내용을 교체해야 하는 **비효율적인 워크플로우**
- **디자인 일관성 유지**가 어렵고, 반복적인 레이아웃 작업에 시간 소요

### 1.4 해결하고자 하는 문제
- 브로슈어 제작을 위한 도구 분산 → **하나의 통합 시스템**으로 해결
- 수동 레이아웃 작업 → **템플릿 기반 자동화**
- 디자인 불일치 → **일관된 브랜드 이미지 유지**

---

## 2. 대상 사용자 (페르소나)

### 2.1 주요 사용자
**여행사 대표/관리자**
- 역할: 상품 기획 및 승인, 브로슈어 최종 확인
- 특징: 시간이 부족하고, 빠른 결과물이 필요
- 목표: 업무 효율화를 통한 본업(영업/상담) 집중

### 2.2 사용 상황
1. **고객 상담 중** - 맞춤 일정표를 빠르게 만들어야 할 때
2. **정기 업데이트** - 기존 상품 정보를 주기적으로 갱신할 때

---

## 3. 핵심 기능 (MVP)

### 3.1 FEAT-2: 일정표 편집기 ⭐ 최우선
> 날짜별 일정, 시간, 장소를 직관적으로 입력하고 편집

**사용자 스토리:**
- 사용자는 날짜별 일정을 드래그앤드롭으로 추가/수정할 수 있다
- 사용자는 각 일정에 시간, 장소, 설명을 입력할 수 있다
- 사용자는 일정을 복사/붙여넣기할 수 있다

### 3.2 FEAT-1: 호텔 정보 관리
> 호텔명, 주소, 사진, 위치 정보 등록/수정

**사용자 스토리:**
- 사용자는 호텔 정보를 등록하고 관리할 수 있다
- 사용자는 호텔 위치를 구글 지도에서 확인할 수 있다
- 사용자는 호텔 사진을 업로드하고 갤러리로 관리할 수 있다

### 3.3 FEAT-3: 이미지 갤러리
> 관광지/호텔 사진 라이브러리 관리

**사용자 스토리:**
- 사용자는 이미지를 카테고리별로 정리할 수 있다
- 사용자는 이미지를 일정표에 쉽게 삽입할 수 있다
- 사용자는 클라우드 스토리지와 동기화할 수 있다

## 8. 제약 사항

- **플랫폼**: 데스크톱 웹 전용 (사무실 PC에서 주로 작업)
