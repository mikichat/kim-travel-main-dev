# 항공 예약 관리 시스템 PRD

## MVP 캡슐

1. 목표: 항공 예약장부 수기 업무를 디지털 자동화하여 마감일 누락을 방지하고 업무 공백을 제로로 만든다
2. 페르소나: 여행사 직원 (김국진 + 여행세상 직원)
3. 핵심 기능: 예약장부 자동화(FEAT-1), 통합 달력 & 알림(FEAT-2), 소통·정산 원스톱(FEAT-3)
4. 성공 지표 (노스스타): NMTL/TL 마감 누락 건수 = 0
5. 입력 지표: 일일 PNR 등록 건수, 알림 확인율
6. 비기능 요구: 큰 글씨 UI (시력 배려), 업무 연속성 보장
7. Out-of-scope: 고객 셀프 체크인 포털, 모바일 앱
8. Top 리스크: 기존 수기 업무에서 디지털 전환 시 적응 기간 중 업무 공백
9. 완화/실험: 점진적 전환 — 수기 장부와 병행 운영 후 안정화되면 전환
10. 다음 단계: /screen-spec으로 화면별 상세 명세 생성

---

## 1. 제품 개요

- **서비스명**: 항공 예약 관리 시스템 (Air Booking Manager)
- **핵심 가치**: 마감일(NMTL/TL/BSP) 누락 방지, 수기 업무 자동화, 업무 공백 제로
- **타겟 사용자**: 여행사 직원 (김국진 + 여행세상 직원 2명 이상)
- **핵심 문제**: 수기 예약장부로 관리 시 NMTL/TL 마감일을 놓치면 예약 자동 취소·금전적 손해 발생

---

## 2. 핵심 기능

### FEAT-1: 항공 예약 관리 허브
- **설명**: PNR 텍스트 입력 → 예약 정보 자동 파싱 → 예약장부 테이블 자동 등록
- **Why**: 수기 입력의 오류와 시간 낭비 제거, 이미 air1에서 PNR 파싱 경험 있음
- **우선순위**: MVP 필수
- **포함 항목**: 고객명(한글/영문), PNR, 항공사/항공편, 구간, 운임, NMTL, TL, 예약 상태, 좌석, 여권정보
- **티켓번호 관리** (발권 후):
  - 탑승객 이름 클릭 → 해당 탑승객의 티켓 목록 표시
  - 1인 다수 티켓 지원 (구간별/재발행 시 추가)
  - 저장 항목: 티켓번호, 발권일, 상태(발권/환불/재발행/VOID)
  - 입력: PNR 파싱 자동 추출 + 수동 입력/수정
  - 보관: 출발일 기준 5년, 이후 자동 삭제
  - 용도: 고객 환불/재발행, 세무/정산 검증, 항공사 분쟁 근거

### FEAT-2: 통합 업무 달력 & 알림
- **설명**: NMTL/TL/BSP 입금일/출발일을 통합 달력에 색상 구분 표시, 데드라인 알림
- **Why**: 마감일 누락이 예약 취소·금전적 손해로 직결 — 가장 간절한 기능
- **우선순위**: MVP 필수 (1순위)
- **포함 항목**: 7일 롤링 대시보드, 월간 달력 뷰, 색상 구분(빨강/노랑/초록), 24시간 전 알림

### FEAT-3: 소통·정산 원스톱
- **설명**: 예약 확정 → 고객 안내문 발송 → 입금 확인 → 발권 → 정산 → 인보이스
- **Why**: 예약 후 정산까지의 전체 흐름을 하나의 시스템에서 관리
- **우선순위**: MVP 2차
- **포함 항목**: 안내문 템플릿, 정산 상태 관리(미수/입금/카드), 인보이스 자동 작성

## 3. 사용자 스토리

- As a 여행사 직원, I want PNR을 붙여넣으면 예약이 자동 등록되길, so that 수기 입력 오류와 시간 낭비를 줄인다
- As a 여행사 직원, I want 발권 후 탑승객 이름을 클릭하면 티켓번호를 조회/관리할 수 있길, so that 환불/재발행/세무/분쟁 시 증빙 자료로 활용한다
- As a 여행사 직원, I want NMTL/TL 마감일이 달력에 자동 표시되고 알림이 오길, so that 마감일을 절대 놓치지 않는다
- As a 여행사 직원, I want BSP 입금일 24시간 전에 알림을 받길, so that 입금을 제때 처리한다
- As a 여행세상 직원, I want 전체 예약 현황을 실시간으로 공유받길, so that 업무 공백 없이 협업한다
- As a 여행사 직원, I want 고객에게 영문이름/좌석 확인 안내문을 원클릭으로 보내길, so that 고객 확인을 빠르게 처리한다

## 4. 제약 조건

- **시간**: 점진적 전환 — 수기 장부와 병행 운영
- **인력**: 김국진 + AI (Claude) 개발, 여행세상 직원은 사용자
- **UI**: 큰 글씨, 눈 편한 UI 필수 (장시간 업무 시력 배려)
- **접근성**: 사무실 LAN 내 접근 (추후 도메인 배포)

## 5. MVP 범위

### Phase 0 (MVP): 예약장부 + 달력 + 알림

- 정산 관리
- 인보이스

### Phase 2: 고객·거래처
- 고객 관리
- 거래처 관리

## 벤치마킹 분석

- **유사 서비스 1**: Travelport (아바쿠스/갈릴레오) — GDS 시스템이지만 예약장부 기능은 부재, 별도 장부 관리 필요
- **유사 서비스 2**: 엑셀/구글 시트 — 범용 스프레드시트로 장부 관리, 알림 기능 없음, 마감일 누락 위험
- **유사 서비스 3**: 기존 main 시스템 — 여행 상품/고객 관리 중심, 항공 예약 전용 기능 부재
- **차별점**: NMTL/TL/BSP 마감일 자동 추적 + 알림은 범용 도구에 없는 항공 특화 기능

## 가설 vs 사실

| 항목 | 가설 | 사실 | 검증 방법 |
|------|------|------|----------|
| PNR 파싱 가능 | 아바쿠스/토파즈 PNR 텍스트에서 자동 파싱 가능 | air1에서 PNR 파싱 기능 구현 완료 | 기존 코드 검증됨 |
| 마감일 알림 효과 | 알림만으로 마감일 누락 방지 가능 | 미검증 | 1개월 운영 후 누락 건수 측정 |
| 여행세상 공유 필요 | 여행세상 직원도 실시간 접근 필요 | 사실 — 역할별 등록/공유 구조 필요 | 사용자 확인 완료 |

## 실패 대응

- **MVP 정의**: 대시보드 + 예약장부 + 달력 + 알림
- **실패 시나리오**: 디지털 전환 시 기존 수기 업무와 이중 작업으로 오히려 부담 증가
- **학습 목표**: 수기→디지털 전환 시 실제로 필요한 최소 기능 파악
- **Pivot 옵션**: 전체 전환 대신 알림 기능만 먼저 운영 (수기 장부 + 달력 알림 하이브리드)
