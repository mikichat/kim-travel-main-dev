Oh-My-OpenCode 에이전트 설정 완전 가이드
설정 파일 구조
~/.config/opencode/
├── opencode.json          ← 프로바이더 정의 (API 엔드포인트, 인증)
└── oh-my-opencode.json    ← 에이전트별 모델 매핑
두 파일이 연동되어야 에이전트가 정상 작동
---
1. oh-my-opencode.json (에이전트 설정)
이 파일에서 각 에이전트가 어떤 모델을 사용할지 정의:
{
  $schema: https://raw.githubusercontent.com/code-yeongyu/oh-my-opencode/master/assets/oh-my-opencode.schema.json,
  google_auth: false,
  auto_update: true,
  sisyphus_agent: {
    planner_enabled: true,
    prompt_append: ... 추가 지시사항 ...
  },
  agents: {
    oracle: {
      model: openai/gpt-5.2,
      temperature: 0.3,
      prompt_append: Cross-validate architectural decisions.
    },
    explore: {
      model: zai-coding-plan/glm-4.7,
      temperature: 0.3,
      prompt_append: Provide thorough, structured analysis.
    },
    librarian: {
      model: kimi-for-coding/kimi-for-coding,
      temperature: 0.3,
      prompt_append: Use autonomous tool-calling for deep research.
    },
    frontend-ui-ux-engineer: {
      model: google/gemini-3-pro-high,
      prompt_append: Create modern, polished UI.
    },
    document-writer: {
      model: minimax/MiniMax-M2.1,
      temperature: 0.3,
      prompt_append: Create clear, well-structured documentation.
    },
    multimodal-looker: {
      model: google/gemini-3-flash,
      prompt_append: Thorough image/PDF/diagram analysis.
    }
  },
  ralph_loop: {

모델 지정 형식
"model": "프로바이더명/모델명"
---
2. opencode.json (프로바이더 정의)
oh-my-opencode.json에서 지정한 프로바이더가 여기에 정의되어야 함.
내장 프로바이더 (설정 불필요)
| 프로바이더 | 처리 방식 |
|-----------|-----------|
| openai/* | opencode-openai-codex-auth 플러그인 |
| google/* | opencode-antigravity-auth 플러그인 |
| minimax/* | oh-my-opencode 내장 |
커스텀 프로바이더 (직접 설정 필요)
GLM-4.7 (Z.AI Coding Plan)
핵심: 엔드포인트에 /coding/ 경로 필수!
zhipu: {
  npm: @ai-sdk/openai-compatible,
  name: Z.AI GLM Coding Plan,
  env: [GLM_API_KEY],
  options: {
    baseURL: https://api.z.ai/api/coding/paas/v4
  },
  models: {
    glm-4.7: {
      name: GLM-4.7 Coding Plan,
      limit: {
        context: 200000,
        output: 128000
      }
    }
  }
}
주의: 
- ❌ https://api.z.ai/api/paas/v4 (일반 API - 느림)
- ✅ https://api.z.ai/api/coding/paas/v4 (코딩 플랜 - 빠름)
oh-my-opencode.json과 매핑
oh-my-opencode.json에서 zai-coding-plan/glm-4.7 사용 시, 
플러그인이 내부적으로 zhipu 프로바이더와 매핑함.
---
3. 환경변수
# ~/.zshrc 또는 ~/.bashrc
export GLM_API_KEY="your-api-key"
# 적용
source ~/.zshrc
---
4. 플러그인 설정 (opencode.json)
{
  plugin: [
    oh-my-opencode,
    opencode-antigravity-auth@1.2.7,
    opencode-openai-codex-auth
  ]
}
| 플러그인 | 역할 |
|---------|------|
| oh-my-opencode | Sisyphus 에이전트, 서브에이전트 오케스트레이션 |
| opencode-antigravity-auth | Google Gemini 모델 인증 |
| opencode-openai-codex-auth | OpenAI 모델 인증 (GPT-5.2, o3 등) |
---
5. 수정하면 안 되는 것들
| 항목 | 이유 |
|------|------|
| "plugin": [...] 순서 | 플러그인 로딩 순서 변경 시 인증 깨질 수 있음 |
| google 프로바이더 직접 정의 | Antigravity가 자동 처리 |
| openai 프로바이더의 "api": "codex" | Codex Auth 의존 |
| minimax 프로바이더 직접 정의 | oh-my-opencode 내장 |
---
6. 작동 확인된 에이전트
| Agent | Model | 상태 |
|-------|-------|------|
| oracle | openai/gpt-5.2 | ✅ Codex Auth |
| explore | zai-coding-plan/glm-4.7 | ✅ 엔드포인트 수정 후 작동 |
| document-writer | minimax/MiniMax-M2.1 | ✅ 내장 지원 |
| multimodal-looker | google/gemini-3-flash | ✅ Antigravity |
---
7. GLM-4.7 수정으로 얻은 개선
| 항목 | 수정 전 | 수정 후 |
|------|---------|---------|
| 엔드포인트 | /api/paas/v4 | /api/coding/paas/v4 |
| 응답 시간 | 2분+ | 12초 |
| 코딩 최적화 | ❌ | ✅ |
---
8. 전체 설정 파일 예시
~/.config/opencode/opencode.json
{
  $schema: https://opencode.ai/config.json,
  mcp: { ... },
  plugin: [
    oh-my-opencode,
    opencode-antigravity-auth@1.2.7,
    opencode-openai-codex-auth
  ],
  provider: {
    zhipu: {
      npm: @ai-sdk/openai-compatible,
      name: Z.AI GLM Coding Plan,
      env: [GLM_API_KEY],
      options: {
        baseURL: https://api.z.ai/api/coding/paas/v4
      },
      models: {
        glm-4.7: {
          name: GLM-4.7 Coding Plan,
          limit: { context: 200000, output: 128000 }
        }
      }
    },
    deepseek: { ... },
    groq: { ... },
    google: { ... },
    openai: { ... }
  }
}
~/.config/opencode/oh-my-opencode.json
{
  $schema: https://raw.githubusercontent.com/code-yeongyu/oh-my-opencode/master/assets/oh-my-opencode.schema.json,
  agents: {
    explore: {
      model: zai-coding-plan/glm-4.7,
      temperature: 0.3
    },
    document-writer: {
      model: minimax/MiniMax-M2.1,
      temperature: 0.3
    }
  }
}

핵심 포인트 요약
1. 두 개의 설정 파일이 연동됨
   - oh-my-opencode.json: 에이전트 → 모델 매핑
   - opencode.json: 프로바이더 → API 엔드포인트
2. GLM Coding Plan: /coding/ 경로 필수
3. 내장 프로바이더: minimax, google, openai는 플러그인이 처리
4. 환경변수: 프로바이더의 env 배열에 명시된 키 export 필요
