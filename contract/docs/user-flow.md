# User Flow — 여행사 계약·견적·일정 자동화 시스템

## 1. 개요

본 문서는 여행사 인트라넷 시스템의 사용자 흐름을 정의합니다.

### 1.1 사용자 유형

| 사용자 유형 | 역할 | 권한 |
|-----------|------|------|
| **실무자** | 견적·계약·일정 작성 담당 | 단체 생성/수정, 문서 생성 |
| **관리자** | 계약 확정 및 전체 관리 | 전체 권한, 상태 확정 |

---

## 2. 주요 사용자 시나리오

### 2.1 시나리오 1: 신규 단체 견적서 작성

```
[실무자 로그인]
    ↓
[메인 대시보드]
    ↓
[신규 단체 생성 버튼 클릭]
    ↓
[단체 기본 정보 입력]
  - 단체명
  - 출발일/도착일
  - 인원수
  - 1인당 요금
  - 계약금
    ↓
[자동 계산 실행] ← 시스템이 자동으로 계산
  - 박수/일수
  - 총액
  - 잔액
  - 잔액 완납일
    ↓
[일정 입력]
  - Day 1, 2, 3... 일정 추가
  - 각 일정의 날짜 자동 계산
    ↓
[취소 규정 입력]
  - 출발일 기준 취소 규정 추가
  - 취소 날짜 자동 계산
    ↓
[포함/불포함 항목 입력]
    ↓
[견적서 PDF 생성 버튼 클릭]
    ↓
[PDF 미리보기]
    ↓
[PDF 다운로드 또는 저장]
```

### 2.2 시나리오 2: 기존 단체 수정 및 계약서 발행

```
[실무자 로그인]
    ↓
[메인 대시보드]
    ↓
[단체 검색]
  - 단체명으로 검색
  - 또는 목록에서 선택
    ↓
[단체 상세 화면]
    ↓
[수정 버튼 클릭]
    ↓
[정보 수정]
  - 출발일 변경 시
    → 모든 일정 날짜 자동 재배치
    → 취소 규정 날짜 자동 재계산
    → 잔액 완납일 자동 재계산
  - 인원수/요금 변경 시
    → 총액/잔액 자동 재계산
    ↓
[변경 사항 저장]
    ↓
[상태 변경: 견적 → 계약]
    ↓
[계약서 PDF 생성]
    ↓
[PDF 다운로드]
```

### 2.3 시나리오 3: 계약 확정 및 문서 이력 관리

```
[관리자 로그인]
    ↓
[메인 대시보드]
    ↓
[단체 선택]
    ↓
[단체 상세 화면]
    ↓
[계약 내용 검토]
    ↓
[상태 변경: 계약 → 확정]
  ⚠️ 확정 후에는 자동 계산 비활성화
    ↓
[통합 PDF 생성]
  - 견적서 + 계약서 + 일정표
    ↓
[문서 이력 확인]
  - 버전별 문서 조회
  - 생성 일시/생성자 확인
```

## 10. 사용자 여정 지도 (User Journey Map)

### 10.1 신규 견적서 작성 여정

| 단계 | 사용자 행동 | 시스템 반응 | 감정 | 개선 포인트 |
|-----|-----------|-----------|------|-----------|
| 1. 로그인 | ID/PW 입력 | 대시보드 표시 | 😐 보통 | 자동 로그인 옵션 |
| 2. 신규 생성 | 버튼 클릭 | 입력 폼 표시 | 🙂 긍정 | - |
| 3. 기본 정보 입력 | 날짜/인원 입력 | 자동 계산 실행 | 😊 만족 | 실시간 계산 |
